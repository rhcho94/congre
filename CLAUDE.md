# Congre — 프로젝트 작업 규칙

이 파일은 Claude Code가 자동으로 읽는 컨텍스트 파일입니다. 채팅 클로드(claude.ai 등)는 자동 로드되지 않으므로 새 세션 시작 시 운영자가 직접 붙여넣어야 합니다.

## 새 세션 시작 시 — 핵심 5개 셋트 (Kickoff)

채팅 클로드든 CC든 새 세션을 시작할 때 다음 5개를 컨텍스트에 올린다. 매번 동일.

1. CLAUDE.md (이 파일 전체)
2. AGENTS.md (위 import) — Next.js 버전 특화 규칙
3. docs/handoff/ 최신 파일 — 직전 세션 인계
4. docs/DECISIONS.md — 도메인 결정 (사고 진단 시 첫 grep 대상)
5. docs/known-issues.md — 알려진 이슈 (재발견 방지)

작업 영역에 따라 필요시 추가:
- docs/PROJECT.md — 현재 스냅샷
- docs/ROADMAP.md — 다음 할 일
- docs/CHANGELOG.md — 작업 이력

## 협업 스타일

- **운영자 프로필**: 1인 비개발자 운영자. 코드 직접 수정하지 않으며 Claude Code와 프롬프트로 협업.
- **세션 컨텍스트**: 새 세션 시작 시 위 "핵심 5개 셋트"를 컨텍스트에 올린다.
- **워크플로**: 계획(분석·권장안 제시) → 사용자 승인 → 실행(코드 작성) 순서 준수. 승인 없이 코드 작성 금지.
- **확인과 실행 분리**: "분석만 보고" 지시 시 파일 수정 금지. 실행은 명시적 승인 후에만.
- **Atomic 커밋**: 한 작업 단위를 한 커밋에. 관련 docs 갱신도 같은 커밋에 포함.
- **YAGNI**: 현재 요구사항에 없는 기능·추상화·에러 핸들링 추가 금지.
- **보고 시 숫자 검증**: 파일 수, 라인 수, 커밋 해시 등 보고 전 실제 값 확인. 추정치 보고 금지.
- **커밋 전 git status 필수**: 의도한 파일만 변경됐는지 확인 후 커밋.
- **응답 스타일**: 개발 용어 처음 등장 시 짧게라도 풀어 설명 (비유 포함). 외부 작업(GitHub, Vercel, 카카오 등)은 메뉴 트리 + 단계별로 안내.
- **응답 톤**: 직설적이되 따뜻하게. 사용자 자책 시 위로 과하게 하지 말고 패턴만 짚어줄 것.

### Kickoff 룰 (새 세션 시작 시 채팅 클로드 운영)

채팅 클로드는 파일을 자동으로 못 읽고, 컨텍스트에 흐릿하게 남은 정보를 "안다"고 착각하는 경향이 있음. 다음 4가지로 방지.

1. **5개 셋트 붙여넣기**: 새 채팅 세션 시작 시 CC에서 5개 셋트(위 Kickoff 항목 참조)를 출력받아 채팅 클로드에 직접 붙여넣는다. 매번 동일.
2. **모르면 모른다고 명시 요청**: 첫 메시지에 "이 5개 셋트 외엔 모르는 것으로 간주해라. 진단·결정 시 모르는 영역이면 명시해라"를 포함한다.
3. **비공개 파일은 형식 먼저 제공**: .env 같은 클로드가 못 읽는 파일은 운영자가 변수 이름 목록을 먼저 제공한다. 값은 진짜 필요할 때만. 클로드가 "존재할 거라 추측한 변수"를 묻는 사고 방지.
4. **두 클로드 교차 검증**: CC 진단이 의심스러우면 채팅 클로드에, 채팅 클로드 결정이 의심스러우면 CC에 검증 요청. 한 클로드는 자기 가설 안에 갇혀 있어 외부 검증이 사고 조기 발견에 효과적.

## 절대 규칙

- 브랜드 "Congre" 표기는 반드시 `src/components/BrandName.tsx` 컴포넌트 사용. 인라인 텍스트로 흩어 쓰지 말 것.
- 변수명·파일명·환경변수·도메인 등 기술 식별자는 소문자 (congre-mvp, congre-three.vercel.app).
- Tailwind는 v4 사용 중이며 **config 파일 없음**. `@import` 방식. tailwind.config.js 만들지 말 것.
- 커밋 메시지는 conventional commits (feat:, fix:, chore:, docs:, refactor:).
- 작업 완료 시 자동으로 commit & push.
- **무료 티어 한계는 유료 플랜으로 해결**: Vercel·GitHub Actions·AWS·SOLAPI·Resend·카카오 등 인프라·플랫폼의 무료 티어 제약(쿼터·throttling·기능 제한)에 걸렸을 때, 우회 방법 검토하지 말고 유료 플랜이 표준 답임을 사용자에게 알리고 진행. 어느 플랜·얼마짜리 가입은 사용자 결정 영역.

## 디자인 토큰 (CSS 변수)

- 배경: `var(--bg)` (#0c0b09)
- Surface: `var(--surface)` (#151310), `var(--surface-2)` (#1e1a13)
- 액센트: `var(--accent)` (#c8892c, 앰버/골드)
- 텍스트: `var(--text)` (#ede8df), `var(--muted)` (#79716a)
- 폰트: `var(--font-display)` (Cormorant Garamond italic), `var(--font-body)` (DM Sans)
- 글로벌 유틸: `.rule`, `.glow-accent`, body::after film grain

## 문서 갱신 의무

작업 끝나면 다음을 같은 커밋에 포함:
- 새 기능 추가 → `docs/CHANGELOG.md`에 한 줄 추가
- 기술 결정 (라이브러리 선택, 아키텍처 변경) → `docs/DECISIONS.md`에 항목 추가
- 새로운 알려진 이슈 / 다음 작업 후보 변경 → `docs/ROADMAP.md` 갱신
- 환경변수 / 기술 스택 / 디자인 토큰 변경 → `docs/PROJECT.md` 해당 섹션 갱신
