## 확정된 정책

### 시간축
- T=0: 주최자가 "마감하기" 버튼을 누르는 시점 (기존 deadlineAt)
- T+E: 시스템이 산정한 예상 완료 시각 (E = 클립 수 기반)
- T+E+30분: 50% 환불 대상 확정선 — 넘으면 무조건 50% (번복 없음)
- T+24시간: 100% 환불 대상 확정선 — 영상 작업은 끝까지 진행

### E 산정 공식 (1차안)
E = max(15분, 클립수 × 30초 + 10분)

- 클립 5개 → 15분
- 클립 20개 → 20분
- 클립 50개 → 35분
- 클립 100개 → 60분

필드 테스트 후 조정 예정. 현재는 s3Keys.length를 클립 수로 사용
(render/start 시점에 클라이언트가 전달).

### 알림 발송 매트릭스

| 시점          | 고객(이메일+SMS) | 사내 담당자(SMS×2) |
|---------------|------------------|--------------------|
| T=0           | 편집 시작 + 예상 완료 안내 | — |
| T+E           | 지연·사과 + 30분 후 50% 환불 사전 고지 | 강하게 |
| T+E+30분      | 50% 환불 대상 확정 | 강하게 (환불 처리 트리거) |
| 렌더 완료     | 영상 + 환불 안내(있으면) | 슬랙/SMS |
| T+24시간 미완 | 100% 환불 + 영상은 완성 약속 | 긴급 |

### 환불 처리 원칙
- 50% 환불은 알림만 발송, 실제 처리는 운영자 수동 (CS 응대 시)
- 100% 환불도 알림 자동, 실제 처리 수동
- 카카오톡 채널 @congre 통해 환불 절차 안내
- 전화 응대 없음 (걸지만 받지 않음)
- 품질 클레임은 받지 않음 (유료 결제 동의서에 명시 — 별도 작업)

### 알림 톤
- 사회성·사과·이해 요청 강조
- 서명: "꽁그레팀 드림" / SMS는 `[Congre]` prefix
- 중간 리마인더 없음 (10분/10분 같은 잦은 알림은 짜증만 유발)

```
expectedCompletedAt: Timestamp       — T+E
refund50At: Timestamp                — T+E+30분
refund100At: Timestamp               — T+24시간

notifications: {
  renderStartedNotifiedAt: Timestamp | null
  renderDelayedNotifiedAt: Timestamp | null
  refund50NotifiedAt: Timestamp | null
  refund100NotifiedAt: Timestamp | null
}

refundStatus: 'none' | 'pending_50' | 'pending_100' 
            | 'processed_50' | 'processed_100'
```
