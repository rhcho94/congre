### 4개 결정사항 (다음 세션 첫 번째 할 일)

다음 세션 시작 시 4가지를 순서대로 결정하고 실행한다.

**[결정 1] 데이터 구조**: Firestore `clips/{clipId}` 문서에 `excluded: boolean` 필드 추가 vs. 별도 컬렉션.
- 추천: `excluded: boolean` 필드. 단순하고 쿼리가 기존 코드와 동형.

**[결정 2] UI 흐름**: 제외 버튼을 어디에?
- A: 클립 목록 각 행에 [제외] 토글 버튼
- B: 클립 클릭 시 팝업/모달에 제외 옵션
- 추천: A (한 화면에서 모든 클립 제어 가능).

**[결정 3] 가역성**: 제외 후 복원 가능?
- A: 단방향 삭제 (Firestore 문서 delete 또는 `excluded: true`)
- B: 양방향 토글 (excluded 필드 true/false)
- 추천: B (실수 복구 가능, 주최자 UX 안전).

**[결정 4] 필터 위치**: `render/start`에서 필터링 vs. `host/clips` API에서 필터링.
- 추천: `render/start`에서 `.where("excluded", "==", false)` 또는 excluded 없는 클립만 포함.
- 단, Firestore composite index 필요할 수 있음 (eventId + excluded 조합). 로컬 에뮬레이터에서 먼저 테스트.

### 예상 구현 범위 (결정 1·3·4를 추천안으로 채택 시)

1. `Clip` 인터페이스 — `excluded?: boolean` 추가 (`src/lib/events.ts`)
2. `PUT /api/clips/[clipId]` 신설 — 호스트 인증 후 `excluded` 업데이트
3. `GET /api/host/clips` — 응답에 `excluded` 필드 포함
4. `POST /api/render/start` — clips 쿼리에 `excluded != true` 조건 추가
5. `src/app/dashboard/events/[eventId]/page.tsx` — 클립 행에 제외 버튼 추가, API 호출
