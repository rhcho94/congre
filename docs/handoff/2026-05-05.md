## 구현 중 확정된 결정 사항

### refundStatus 타입 단순화
- 2026-05-04.md 핸드오프에는 `'pending_50' | 'pending_100' | 'processed_50' | 'processed_100'`로 설계했으나 구현 시 `'none' | '50' | '100'`으로 단순화.
- 근거: pending/processed 구분은 YAGNI — 현재 환불 처리는 수동이므로 상태 전이가 불필요.
- 실제 Firestore 필드값: `"none"` (초기) → `"50"` (refund_50At 초과 시) → `"100"` (refund100At 초과 시).

### 크론 쿼리 단일 필터
- `status == "rendering"` 단일 조건만 사용, 시간 비교는 앱 코드에서 처리.
- 근거: Firestore 복합 인덱스 추가 없이 배포 가능.

### dashboardUrl 구성 (크론 내부)
- `process.env.NEXT_PUBLIC_APP_URL ?? ""` 사용 (request.headers.get("origin") 불가 — scheduled context).
- NEXT_PUBLIC_APP_URL: `https://congre-three.vercel.app` (Vercel 등록 완료).

### 레거시 이벤트 스킵
- cron에서 `expectedCompletedAt` 필드가 없는 이벤트는 `continue` — [1] 이전에 시작된 렌더는 새 정책 대상 아님.

### 테스트 라우트 삭제 (hotfix)
- `src/app/api/test/notify-render-started` — Phase 1 검증용으로 추가됐으나 RenderStartedCtx 변경 후 미동기화 상태로 빌드 실패 유발.
- 수정 대신 삭제: 이미 검증 완료(5/5) + renderEstimateMin 공식 이중 유지는 DRY 위반.

### GitHub Actions cron throttling
- `* * * * *` (매분) 스케줄은 GitHub free tier throttling으로 약 4시간에 1회만 실행됨(관측).
- `*/5 * * * *` (5분 간격)으로 조정. 1주일 재관측 예정.
