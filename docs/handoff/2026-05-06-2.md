## Phase B-3 2단계 완료 내용

### 신설 API 라우트

| 라우트 | 역할 |
|--------|------|
| `GET /api/events` | 호스트 이벤트 목록 (Bearer auth, createdAt desc) |
| `GET /api/host/events/[eventId]` | 호스트 이벤트 상세 (Bearer auth, hostId 소유권 검증) |
| `GET /api/host/clips?eventId=` | 호스트 클립 목록 (Bearer auth, hostId 소유권 검증) |

반환 필드: 모두 Timestamp → milliseconds(number). sessionToken, organizerEmail 등 민감 필드 제외.

### 대시보드 변경

- `src/app/dashboard/page.tsx`: `subscribeToHostEvents` → `GET /api/events` polling (탭 숨김 시 중단)
- `src/app/dashboard/events/[eventId]/page.tsx`:
  - `subscribeToEvent` → `GET /api/host/events/[eventId]` polling (5초, 탭 숨김 시 중단)
  - `subscribeToClips` → `GET /api/host/clips?eventId=` polling (5초, 탭 숨김 시 중단)
- `src/lib/events.ts`: `subscribeToEvent`, `subscribeToHostEvents`, `subscribeToClips` 3함수 삭제. 가져오기에서 `onSnapshot`, `query`, `where` 제거.

### render/start 변경

- `POST /api/render/start` body: `{ eventId, s3Keys, eventTitle? }` → `{ eventId }` 만
- 서버에서 Admin SDK로 clips 직접 fetch
- 빈 clips 가드: `if (clipsSnap.empty) return 400 { error: "NO_CLIPS" }`
- 대시보드 호출부 body도 `{ eventId }` 만으로 동기화

### Firestore rules (로컬 + 커밋 완료)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /events/{eventId} {
      allow read: if false;
      allow create: if request.auth != null;
      allow update: if request.auth != null && request.auth.uid == resource.data.hostId;
    }
    match /clips/{clipId} {
      allow read, create: if false;
      allow update, delete: if request.auth != null;
    }
    match /notifications/{any} {
      allow read, write: if false;
    }
  }
}
```
