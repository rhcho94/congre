## 버그 수정 상세

### 1. handleClose silent fail (`5c4559a`)

**발견 경위**: 운영자가 모든 클립 제외 후 마감 시도 → 경고 없이 status="closed", 영상 없음 상태로 정지.

**근본 원인**: `fetch()`는 non-2xx 응답에서 throw하지 않음. 기존 코드는 응답 자체를 읽지 않았음. `render/start`의 Firestore 업데이트(`status: "rendering"`)는 early return 이후에 위치하므로 status는 "closed"로 남음.

**수정**: `res.ok` 체크 + 4-way 에러 코드 분기:
- `NO_CLIPS_AFTER_EXCLUSION` → "선택된 클립이 없어요. 제외를 해제한 뒤 페이지에서 다시 시도해주세요."
- `NO_CLIPS` → "업로드된 클립이 없어요."
- `NOT_CONFIGURED` → "서버 설정 오류..."
- 기타 → `영상 생성 시작에 실패했습니다 (코드/상태코드)...`

**잠재성**: 클립 제외 기능 이전부터 존재. 503/500 등도 동일하게 silent fail이었으나 이번에 처음 표면화.

**운영 메모**: 사고로 생긴 어정쩡한 상태 이벤트(status=closed + renderStartedAt 없음)는 자동 복구 경로 없음. CS(@congre 카카오톡) 응대 후 Firestore 직접 정리.

### 2. [영상 생성 다시 시작] 버튼 (`170089c`)

**발견 경위**: silent fail 픽스 후 "제외를 해제하세요" alert는 뜨지만 해제 후 재시도 경로가 없음.

**도메인 제약**: 참가자 100명 재업로드 요청 불가 → retry path 필수.

**수정**:
- `callRenderStart(idToken, eventId): Promise<boolean>` DRY 추출 (handleClose + handleRestartRender 공유)
- `handleRestartRender`: closing state 재사용 (modal과 버튼은 status 조건으로 상호 배타적)
- 버튼 노출 조건: `event.status === "closed" && clips.length > 0`
- `render/start`에 `event.status` 체크 없음 → "closed" 이벤트에서 재호출 가능
